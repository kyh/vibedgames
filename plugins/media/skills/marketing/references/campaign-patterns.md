# Marketing campaign patterns

Use these patterns to turn a request into an asset matrix before generating.

## Hook, proof, context, conversion

The default paid-social set.

| Role | Purpose | Asset shape |
| --- | --- | --- |
| Hook | Stop the scroll | bold first frame, problem image, surprising product detail |
| Proof | Make it believable | demo, process, feature, result, screenshot, texture |
| Context | Make it relevant | audience situation, lifestyle, workflow, environment |
| Conversion | Make action easy | clean final frame with safe space for external CTA |

## Awareness ladder

Match the asset to audience awareness.

| Audience state | Angle |
| --- | --- |
| Unaware | name the problem visually, no heavy product detail |
| Problem-aware | show the frustrating moment and consequence |
| Solution-aware | compare approach, workflow, or category alternatives |
| Product-aware | show product detail, feature, proof, or demo |
| Ready to act | clean offer frame, user-supplied CTA space |

## Channel matrix

| Channel | Useful deliverables |
| --- | --- |
| TikTok/Reels/Shorts | 9:16 hook video, creator variant, product b-roll |
| Meta paid social | 1:1 and 4:5 stills, 9:16 video, retargeting proof |
| Landing page | hero image, product detail, comparison visual, feature cards |
| Email | header image, product still, feature/detail graphic |
| Display | wide banner safe zones, simple product and benefit visual |
| App store/Product page | clean screenshots, feature visualization, icon-safe crops |

## Variation axes

Vary one axis at a time.

- Audience: beginner, expert, buyer, operator, creator, founder.
- Problem: time, cost, complexity, trust, quality, status, risk.
- Proof: demo, screenshot, texture, testimonial, process, comparison.
- Setting: desk, bathroom, kitchen, gym, store, studio, outdoor, event.
- Tone: direct, expert, warm, premium, playful, practical, urgent.
- Format: still, video, creator, carousel, before-after, product detail.

## Claim discipline

Never invent:

- review quotes
- user counts
- revenue or conversion numbers
- clinical, financial, legal, safety, or performance claims
- partner logos, award badges, app store ratings, press logos

When claims are missing, use observable language: "shows the workflow",
"cleaner setup", "fewer steps on screen", "product texture visible".
