# RESOURCES.md Format

`RESOURCES.md` is the curated set of trusted sources for learning game development. Knowledge for lessons should be drawn from here, not from parametric guesses. Wisdom comes from the communities and playtest channels listed here.

## Structure

```md
# Game Development Resources

## Knowledge

- [Book: _Game Feel_ — Steve Swink](https://example.com)
  The canonical text on why controls feel good or bad. Use for: input response, juice, polish lessons.
- [Docs: Phaser API documentation](https://docs.phaser.io)
  Primary engine reference. Use for: any engine API question — check here before answering from memory.
- [Video: "The Secret of Good Game Feel" — GMTK](https://example.com)
  Tight visual introduction to hit stop and screen shake. Use for: the juice lesson's recommended primary source.

## Wisdom (Communities & Playtesting)

- [Phaser Discord](https://discord.gg/phaser)
  Official engine community. Use for: engine questions beyond the docs, code review from practitioners.
- [GMTK Game Jam](https://itch.io/jam/gmtk-jam)
  Annual jam with a huge feedback culture. Use for: a deadline that forces shipping, structured peer feedback.
- Playtest channel: deployed game links shared to {friends/family/colleagues}
  Use for: watching real players hit the first 30 seconds cold.
```

## Rules

- **High-trust only.** Prefer primary sources (engine docs), recognised practitioners, and well-moderated communities. If a resource is marketing dressed as education, leave it out.
- **Annotate every entry.** A bare link is useless in three months. Add one line: what it covers and when to reach for it.
- **Group by Knowledge / Wisdom.** Mirrors the philosophy in [SKILL.md](./SKILL.md). It is fine for a resource to appear in only one group.
- **Surface gaps explicitly.** If no good resource exists for an area the mission needs, write a `## Gaps` section listing what is missing. This drives future search.
- **Prune ruthlessly.** A resource that turned out to be wrong, shallow, or off-mission should be removed, not buried. Better five sharp sources than thirty mediocre ones.
- **Record community preferences.** If the user has opted out of joining communities, note it here so future sessions don't keep proposing them — and lean on playtesting instead.
